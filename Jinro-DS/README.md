# Jinro DS Logbook

人狼ゲーム中の行動を選択式で記録し、自動で特徴量テーブルに集計するUIです。
初日は「誰が、誰に対して、どのように、どんな行動をする」を記録し、
`features.txt` の定義に合わせて出力します。

## 使い方

1. プレイヤー名簿をカンマ/空白区切りで入力し「名簿を登録」。
2. 役職(真相)は人狼/占い師/騎士/狂人を選択し、残りを村人として登録。
3. 盤面は占いCO数・霊媒CO数を1〜13から選択し、`m-n` 形式で記録。
4. 行動ログを「誰が / 誰に / 行動 / 理由」で追加。
5. 状態を書き出し/読み込みして作業を再開できます。
6. 特徴量テーブルをCSV.gz/JSONで書き出し。

## 特徴量の対応

- `shikou_koushin_count_1`: 思考更新の回数
- `shiroage_*_1`: 白上げの対象人数 (感覚/論理)
- `utagai_*_1`: 疑いの対象人数 (感覚/論理)
- `binjou_1`: 議論中の便乗有無 (0/1)
- リーク特徴量: 役職が入力されている場合に計算

## Dockerで起動

### 変更をリビルドなしで反映する方法

```bash
docker run --rm -p 8080:80 \
  -v "$(pwd)":/usr/share/nginx/html:ro \
  nginx:1.27-alpine
```

`http://172.25.44.91:8080` にアクセスし、ファイルを編集したらブラウザをリロードしてください。

### 固定イメージとしてビルドする方法

```bash
docker build -t tablemaker .
docker run --rm -p 8080:80 tablemaker
```

## テスト

```bash
# 簡易チェック (HTML/JS/CSS が存在するか確認)
ls -1 index.html styles.css app.js
```

## データの入出力

- 状態JSON: プレイヤー/役職/盤面/備考/行動ログを保存・復元します。
- CSV.gz: 先頭行をヘッダーとしてgzip圧縮で書き出します。
- JSON: 以下の形式で書き出します。

```json
{
  "columns": ["player", "role", "banmen", "bikou", "..."],
  "rows": [
    ["Alice", "村人", "2-1", "", "..."],
    ["Bob", "人狼", "2-1", "", "..."]
  ]
}
```
